## Step-by-step

After completing the steps below, you will have a local development environment where you can learn about the anatomy of a dropin ([Dropin Anatomy](/quick-start/dropin-anatomy/)) and find the resources available ([Dropin Resources](/quick-start/dropin-resources/)) to begin your Commerce storefront journey.

<Steps>

### Create your storefront repository

Navigate to [aem-boilerplate-commerce](https://github.com/hlxsites/aem-boilerplate-commerce).
Click the **Use this template** button and select **Create a new repository**.
Choose an owner and repository name. Optional: Choose a description and visibility, then click the **Create repository** button.

### Clone your storefront repository

Navigate to your new repository and click the **Code** button.
Copy the URL (HTTPS or SSH).
Open your terminal and navigate to the directory where you want to clone your storefront repository.
Run the following command, replacing `URL` with the URL you copied:

```bash frame="none"
git clone URL
```

### Install storefront dependencies

Navigate to your new storefront directory and run the following command:

```bash frame="none"
npm i
```

### Add the AEM Code Sync app

Navigate to the [AEM Code Sync app](https://github.com/apps/aem-code-sync) and click the **Configure** button. Select the organization account to add the app to. From GitHub's AEM Code Sync page, in the **Repository access** section, select the repository you created and click the **Save** button. If the installation was successful, you'll be redirected to a success screen on https//aem.live. You can close this browser tab.

### Start the local development server

Run the following command:

```bash frame="none"
npm start
```

You should see the first page of your boilerplate storefront at `http://localhost:3000`.

</Steps>
