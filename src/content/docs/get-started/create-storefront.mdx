The quickest path to a new storefront starts with our [Commerce boilerplate](https://github.com/hlxsites/boilerplate-commerce-dropins). So let's use it to create a starter storefront using Edge Delivery Services and Adobe Commerce dropin components built-in.

## Step-by-step

After completing the steps below, you will have the local development environment you need to start developing your storefront.

<Tutorial>

### Create your storefront repo

1. Navigate to [aem-boilerplate-commerce](https://github.com/hlxsites/aem-boilerplate-commerce).
1. Click the **Use this template** button and select **Create a new repository**.
1. Choose an owner and repository name.
1. Add a description and choose a visibility (we recommend public).
1. Click the **Create repository** button and watch GitHub create your new repo.

**What happens?** When you use the template button (instead of cloning), GitHub copies all the content from the boilerplate repo (files and folders), but none of the commit history, remote links, or other version-control files. This ensures that your new storefront repository starts with a clean slate, ready for new development.

### Clone your storefront locally

1. In your new GitHub repository, click the **Code** button and copy the repo's URL (HTTPS or SSH).
1. Open a terminal and run the following command, replacing `URL` with the URL of your storefront repo.

   ```bash frame="none"
   git clone URL
   ```

### Install storefront dependencies

1. Navigate to the root of your local storefront project and run the following command:

   ```bash frame="none"
   npm i
   ```

### Add the AEM Code Sync app

1. Navigate to the [AEM Code Sync app](https://github.com/apps/aem-code-sync) and click the **Configure** button.
1. Select the organization account to add the app to.
1. From GitHub's AEM Code Sync page, in the **Repository access** section, select the repository you created and click the **Save** button.
1. If the installation was successful, you'll be redirected to a success screen on https://aem.live. You can close this browser tab and proceed.

### Start the development server

1. Run the following command:

   ```bash frame="none"
   npm start
   ```

You should see the first page of your boilerplate storefront at `http://localhost:3000`.
